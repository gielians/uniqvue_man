Sessions

Die Session List

Das Session List Fenster unterteilt sich in 3 Spalten. Ganz Links befinden sich vertikal angeordnet die Bearbeiten Icons (Konsistente Bedienelemente), darauf folgt in der Mitte das Hauptfenster mit der Listen Ansicht der Sessions und ganz rechts der Session Inspector.

Die Session List hat mehrer Reihen die dazu dienen die Session List zu filtern. In der Liste selber übernehmen sie aber je nach Typ weitere Eigenschaften wie folgt.

  1. Name - In der Liste kann ich durch doppelklick einen individuellen Namen vergeben. Dieser Name wird beim bestätigen automatisch gespeichert.
  2. Category - Hier können sie ihrer Session eine Kategorie vergeben. Die Kategorien finden sich auch als Ordner im Output wieder wo sie Ihre Session so einfacher wiederfinden können.
  3. Date - Beim erstellen einer Session wird automatisch das aktuelle Datum vergeben, das hilft ihnen die Sessions nach Datum zu sortieren und wieder zu finden.
  4. Author - Das ist sozusagen ihre Signatur und lässt sie nach dem Author einer Session filtern.
  5. Session Type - Eine Session kann einem bestimmten Typ angehören wie Workshop oder Präsentation. Haben sie einen Typ vergeben ist es einfach alle Workshops zu filtern und anzeigen zu lassen.
  6. Released - Sobald sie einen Haken gesetzt haben ist die Session im Output mit ihren Inhalten verfügbar.
  7. Preload - Haben sie den Haken hier gesetzt wird die Session vorgeladen. Somit verkürzen sie Ladezeit beim aufrufen im Output.
  8. Toolbox - Hier können sie eine Toolbox auswählen die dann in der Session zu Verfügung steht. Was eine Toolbox ist erfahren sie im Kapitel Toolboxes. Wie sie eine Toolbox erstellen und konfigurieren erfahren sie im Kapitel Toolbox Workflow.
  9. Style - Wählen sie hier einen Style für die Session aus. Was genau ein Style ist finden sie im Kapitel Styles. Wie sie einen Style erstellen und konfigurieren können finden sie im Kapitel Style Workflow.
  10. Default Background - Legen sie hier einen der 3D Backgrounds für die Session fest. Dieser Background gilt dann für alle Module in der Agenda. Sie können natürlich für jeden Agenda Punkt einen eigene Background festlegen. Wie das genau geht finden sie im Kapitel Session Workflow. Was ein Modul ist finden sie im Kapitel Module. Dort werden auch alle Module und was sie können beschrieben.

Im Session Inspector sehen sie erst etwas wenn sie aus der Liste eine Session selektiert haben. Von oben nach unten finden sie dann folgende Informationen

Session: Den Namen den sie vergeben haben

Author: Wer die Session erstellt hat

Default Background: Ein Thumbnail des ausgewählten Backgrounds

Session Description: Eine Notiz die sie der Session anheften können

Und zu guter letzt den Edit Session Button mit dem sie auch die Session bearbeiten können. Wie das geht und was sie dort alles machen können finden sie im Kapitel Session Workflow.
